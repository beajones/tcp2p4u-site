tcp2p4u

Lightweight, modular, peer-to-peer broadcast messaging.

Functionality

  • Broadcast messsaging - send messages to a group of peers at once
  • Command line interface - send messages while performing other tasks on the command line without having to refresh
  • Lightweight - no database limitations, library dependencies, or unnecessary additions
  • Distributed - being peer-to-peer makes the system fault-tolerant and censorship-resistant
  • Throwaway sessions - once the session closes, chat logs are wiped clean

Glossary

Request Commands

  • start - takes no arguments; creates a locally hosted Peer.
  • broadcast - takes a variable number of arguments, joined by semicolons; sends as a message to every Peer to which the locally hosted Peer is connected.
  • join - takes a variable number of arguments; instructs the locally hosted Peer to connect to remote Peers at given IPs.
  • info - takes no arguments; prints out basic information about the locally running Peer.
  • chat - takes no arguments; prints out the current chat.
  • stop - takes no arguments; terminates the locally hosted Peer.

Set-up Commands

  • Run make to compile.
  • Run make starter to compile and create a starter Peer.
  • Run make clean to delete class files.
  • Run java Peer X & to create a Peer with a PeerID of X.
  • Run java PeerRequester with command line arguments to make requests to your locally hosted Peer.
